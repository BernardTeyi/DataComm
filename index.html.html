What is Data Communication?

Data communication is the exchange of data between two or more devices through a transmission medium — cable or wireless. Every communicating device must follow the same rules, called a protocol, to ensure data is correctly sent, received, and understood.

The study of data communication covers physical signals, transmission media, network protocols, error detection, and the architectural models (OSI, TCP/IP) that organise these into manageable layers.

The 5 Components of Every Communication System
Component 1

📤 Sender

The device that originates the data — a laptop, phone, sensor, or server initiating communication.

Component 2

📨 Message

The information being transmitted — text, numbers, images, audio, video, or any digital data.

Component 3

📡 Medium

The physical or wireless path — copper wire, fibre optic cable, or electromagnetic radio waves.

Component 4

📥 Receiver

The destination device that interprets and reconstructs the incoming signal into usable data.

Component 5

📋 Protocol

Agreed rules governing format, timing, error-checking, and acknowledgement between devices.

How Data Travels From A to B
1

Encoding

The sender converts the message into a suitable signal — electrical pulses, light pulses, or radio waves depending on the medium.

2

Transmission

The signal travels through the medium. Distance, interference, and noise can degrade it — error-detection mechanisms identify problems at this stage.

3

Decoding & Reception

The receiver converts the incoming signal back into data and verifies integrity using CRC, parity bits, or checksums before delivering it to the application.

// 03 — Core Concepts

Core Concepts

The fundamental ideas that underpin all data communication systems — from signals and modes of transmission to the protocols that keep data intact and orderly.

Transmission Modes

The transmission mode defines the direction in which data flows between two communicating devices.

Simplex

One Direction Only

Data flows in one direction only. The sender can only send; the receiver can only receive. No reverse channel exists.

Examples: Keyboard → CPU, television broadcast, FM radio
Half-Duplex

Both Directions, Not Simultaneously

Data flows in both directions but not at the same time. One device must finish transmitting before the other can respond.

Examples: Walkie-talkies, CB radio, older network hubs
Full-Duplex

Both Directions Simultaneously

Data flows in both directions at the same time. Both parties can send and receive concurrently — maximum efficiency.

Examples: Telephone, video calls, modern Ethernet, Wi-Fi
Signal Types

Data is physically carried by either analog or digital signals, each with distinct characteristics and best-use scenarios.

📊 Analog Signals

Continuous, smooth waveforms that vary in amplitude and frequency. They can represent an infinite range of values. Susceptible to noise accumulation but can travel long distances without regeneration.

Examples: Sound waves, AM/FM radio, original telephone systems, seismic signals.

💻 Digital Signals

Discrete, on/off pulses representing binary values (0 and 1). Far more resistant to noise — repeaters can regenerate a clean signal. Easier to process, store, encrypt, and compress.

Examples: Computer data, fibre optic light pulses, Ethernet, digital telephony (VoIP).

Protocols & Error Detection

A protocol is a set of agreed rules governing how data is formatted, transmitted, received, and acknowledged. Without protocols, devices from different manufacturers could not communicate.

Syntax

Format & Structure

Defines the exact format of data — field order, length, and encoding. Both sender and receiver must agree on how data is laid out.

Semantics

Meaning & Action

Defines what each section of data means and what action should be taken. For example, a specific bit pattern might signal "request connection."

Timing

Speed & Sequencing

Controls when and how fast data is sent — handling cases where sender and receiver operate at different speeds through flow control mechanisms.

Error Detection Methods
MethodHow it WorksStrengthUsed In
Parity BitAppends a single bit to make the count of 1s even or oddWeak — detects single-bit errors onlySimple serial communication
ChecksumSends the arithmetic sum of data segments; receiver recalculates and comparesModerate — misses some multi-bit errorsTCP/IP, UDP headers

Hamming CodeAdds multiple parity bits at specific positions to locate and correct errorsCan correct single-bit errors, detect two-bitMemory systems, satellite data
// 04 — Network Models

Network Models

The OSI and TCP/IP models define how network communication is broken into distinct layers — each with a specific role, communicating only with adjacent layers.

The OSI Model — 7 Layers

The Open Systems Interconnection (OSI) model — developed by ISO — is the definitive conceptual framework for understanding network communication. It separates concerns across 7 layers so that each can evolve independently. Click any layer to expand it.

Layer 7ApplicationHTTP · HTTPS · FTP · SMTP · DNS▼
Role
The interface between the network and end-user software. Provides services like file transfer, email, and web browsing directly to applications — the layer users interact with.
Protocols
HTTP, HTTPS, FTP, SMTP, DNS, POP3, IMAP, DHCP, SSH, Telnet
Example
Your browser sends an HTTP GET request to a web server's IP address to load a page.
Layer 6PresentationSSL/TLS · JPEG · MPEG▼
Role
Translates, encrypts, and compresses data so the application layer can understand it regardless of how the other system represents it internally.
Protocols/Formats
SSL/TLS (encryption), JPEG, PNG, MPEG, ASCII, Unicode, GIF
Example
SSL/TLS encrypts your password and payment data before it leaves your device.
Layer 5SessionNetBIOS · RPC · SIP▼
Role
Establishes, manages, and terminates communication sessions between two applications. Handles authentication, authorisation, and reconnection after interruption.
Protocols
NetBIOS, RPC, PPTP, SIP, NFS
Example
A video call maintains a persistent session. If the connection drops briefly, the session layer attempts to restore it without restarting the call.
Layer 4TransportTCP · UDP▼
Role
Provides end-to-end data delivery, error recovery, and flow control. Segments large messages into smaller units and reassembles them in correct order at the destination.
Protocols
TCP (reliable, connection-oriented, three-way handshake) · UDP (fast, connectionless, no acknowledgement)
Example
TCP ensures every packet of a file download arrives, in order. If a packet is lost, TCP requests a retransmission automatically.
Layer 3NetworkIP · ICMP · OSPF · BGP▼
Role
Handles logical addressing (IP addresses) and routing — determining the optimal path for packets to travel across multiple different networks between source and destination.
Protocols
IPv4, IPv6, ICMP, OSPF, BGP, RIP, EIGRP
Example
A router at Layer 3 examines the destination IP address of each packet and decides which path to forward it along using routing tables.
Layer 2Data LinkEthernet · ARP · PPP▼
Role
Node-to-node data transfer between directly connected devices on the same network segment. Uses MAC addresses to identify devices. Handles framing and error detection for physical transmission.
Protocols
Ethernet (IEEE 802.3), Wi-Fi (IEEE 802.11), ARP, PPP, HDLC, 802.1Q (VLAN)
Example
A network switch reads the destination MAC address of an incoming frame and forwards it only to the correct port — not broadcast to all devices.
Layer 1PhysicalEthernet cable · Fibre · Radio▼
Role
Transmits raw binary bits (0s and 1s) over a physical medium. Defines electrical signal voltages, cable specifications, pin layouts, connector types, and data rates.
Examples
Ethernet cables (Cat5e/6/6A), fibre optic, radio waves, hubs, repeaters, NICs, DSL modems
Example
An Ethernet cable physically carries voltage pulses between a PC's NIC and a switch port at 1 Gbps.
The TCP/IP Model — 4 Layers

The TCP/IP model (Internet Model) is the practical implementation that actually runs the internet. It condenses OSI's 7 layers into 4, focusing on what is needed for real-world network communication.

Layer 4 — Top

Application Layer

Combines OSI Layers 5, 6, and 7. All high-level protocols used by applications — handles presentation and session management internally.

HTTP · FTP · DNS · SMTP · DHCP · SSH · Telnet
Layer 3

Transport Layer

Same as OSI Layer 4. Manages end-to-end delivery and data integrity between host applications.

TCP (reliable) · UDP (fast)
Layer 2

Internet Layer

Corresponds to OSI Layer 3. Handles logical IP addressing and routing packets across different networks worldwide.

IP (v4/v6) · ICMP · ARP · OSPF
Layer 1 — Bottom

Network Access Layer

Combines OSI Layers 1 and 2. Handles physical transmission and link-level framing for the local network.

Ethernet · Wi-Fi · Fibre · DSL · PPP

TCP vs UDP

🔗 TCP — Transmission Control Protocol

Connection typeConnection-oriented (3-way handshake)
ReliabilityGuaranteed delivery & correct ordering
Error checkingYes — acknowledgements & retransmission
Flow controlYes — windowing mechanism
SpeedSlower (overhead of acknowledgements)
Use casesHTTP/S, email, file transfers, banking, SSH

⚡ UDP — User Datagram Protocol

Connection typeConnectionless (no handshake)
ReliabilityNo guaranteed delivery or ordering
Error checkingBasic checksum only — no retransmission
Flow controlNone
SpeedFaster (minimal overhead)
Use casesVideo streaming, VoIP, gaming, DNS, DHCP

Network Topologies
🚌

Bus

All devices share a single backbone cable. A break anywhere disrupts the whole network. Simple and cheap but largely obsolete for modern LANs.

⭐

Star

All devices connect to a central switch or hub. Most common LAN topology today — one device failing doesn't affect others. Easy to troubleshoot.

🔄

Ring

Devices form a closed loop. Data travels in one direction. A single failure can disrupt the whole network unless a dual-ring is used (FDDI).

🕸️

Mesh

Every device connects to every other device. Extremely fault-tolerant but expensive. Used in critical infrastructure, WANs, and the internet backbone.

🌳

Tree (Hierarchical)

A hierarchy of star-topology groups connected to a root. Scalable — used in large campus and corporate networks. Failure higher up affects more nodes.

🔀

Hybrid

Combines two or more topologies. Most real-world enterprise networks are hybrid — e.g. a star layout within each floor connecting via a ring backbone.
